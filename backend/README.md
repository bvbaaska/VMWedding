# VMWedding — бэкенд + Telegram-бот

Собирает заявки RSVP с сайта, сохраняет в SQLite, шлёт уведомления в Telegram
и умеет выгружать список гостей в Excel прямо в чат.

## Что приходит в Telegram

При каждой отправке формы админам падает сообщение:

```
💌 Новая заявка с сайта!

ФИО: Иванов Иван
Придёт: ✅ Да, смогу
Комментарий: будем вдвоём с супругой

№3 · 2026-06-07 21:14:02
```

Под сообщением — кнопки:
- **📥 Скачать список гостей (Excel)** — присылает .xlsx со всеми заявками
  (столбцы: №, ФИО, Сможете прийти, Комментарий, Дата отправки)
- **📊 Статистика** — сводка (всего / придут / не уверены / не смогут)

Команды бота: `/start`, `/menu`.

---

## 1. Создать бота и узнать chat_id

1. В Telegram напишите **@BotFather** → `/newbot` → получите **BOT_TOKEN**.
2. Напишите **@userinfobot** → он пришлёт ваш **chat_id** (число).
   Можно указать несколько ID через запятую (жених + невеста).
3. **Важно:** откройте своего бота и нажмите **Start**, иначе он не сможет
   вам писать.

## 2. Локальный запуск (для теста)

```bash
cd backend
python -m venv venv
venv\Scripts\activate        # Windows  (или: source venv/bin/activate)
pip install -r requirements.txt

set BOT_TOKEN=ВАШ_ТОКЕН            # Windows (или export ... на Linux/Mac)
set ADMIN_CHAT_IDS=ВАШ_CHAT_ID
python bot.py
```

Проверка: открыть http://localhost:8080/health → `{"ok": true}`.
Напишите боту `/menu` — должно прийти меню с кнопками.

## 3. Деплой в интернет (чтобы сайт мог слать заявки)

GitHub Pages — статичный, поэтому бэкенд нужно поднять отдельно. Самое простое —
бесплатные/дешёвые хостинги:

### Вариант A — Render.com (просто)
1. Залейте папку `backend/` в отдельный репозиторий (или этот же).
2. Render → **New → Web Service** → подключите репозиторий.
3. Build command: `pip install -r requirements.txt`
   Start command: `python bot.py`
4. В **Environment** добавьте `BOT_TOKEN`, `ADMIN_CHAT_IDS`, `ALLOW_ORIGIN=https://vmwed.ru`.
5. После деплоя получите адрес вида `https://vmwedding.onrender.com`.

### Вариант B — Railway / свой VPS
Аналогично: задать переменные окружения и запустить `python bot.py`
(на VPS лучше через `systemd` или `pm2`, либо `gunicorn` для Flask + поток бота).

> Заявки хранятся в файле `guests.db` (SQLite). На бесплатных хостингах с
> «засыпающим» диском файл может обнуляться — для постоянного хранения возьмите
> план с persistent-диском или подключите внешнюю БД.

## 4. Подключить сайт к бэкенду

В `index.html` найдите строку:

```js
const RSVP_ENDPOINT = ""; // ← сюда адрес бэкенда
```

и впишите адрес своего сервиса, например:

```js
const RSVP_ENDPOINT = "https://vmwedding.onrender.com/api/rsvp";
```

Запушьте — форма начнёт отправлять заявки на сервер и в Telegram.
Пока поле пустое, форма просто показывает «Спасибо» без отправки (демо-режим).
