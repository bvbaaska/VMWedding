# VMWedding — PHP-бэкенд (для хостинга reg.ru)

Версия на PHP — работает на обычном (shared) хостинге через панель управления,
**без SSH и без постоянных процессов**. Telegram-бот работает по webhook.

## Файлы
- `config.php` — настройки (токен, ID админов, секрет)
- `lib.php` — общая логика (хранилище, Telegram, генерация .xlsx)
- `api.php` — приём заявок с формы сайта
- `webhook.php` — обработка команд и кнопок бота
- `setup_webhook.php` — одноразовая привязка webhook (потом удалить)

## Шаг 1. Бот и chat_id
1. @BotFather → `/newbot` → скопируйте **токен**.
2. @userinfobot → пришлёт ваш **chat_id**.
3. Откройте своего бота и нажмите **Start**.

## Шаг 2. Настроить config.php
Откройте `config.php` и впишите:
```php
define('BOT_TOKEN', '123456:AA...');     // ваш токен
$ADMIN_CHAT_IDS = ['123456789'];          // ваш chat_id (можно несколько)
define('WEBHOOK_SECRET', 'любая-строка'); // придумайте секрет
define('ALLOW_ORIGIN', 'https://vmwed.ru');
```

## Шаг 3. Загрузить на хостинг
Через **Файловый менеджер** в панели reg.ru (или по FTP) создайте папку,
например `public_html/api/`, и загрузите туда все файлы из `backend-php/`.

Требования к хостингу (есть почти всегда): PHP 7.4+, расширения **cURL** и **Zip**.
Папка должна быть доступна на запись (для файла `guests.jsonl`).

## Шаг 4. Привязать webhook
Откройте в браузере один раз:
```
https://vmwed.ru/api/setup_webhook.php
```
Должно показать `"ok": true`. Напишите боту `/menu` — придёт меню.
После этого **удалите** `setup_webhook.php` с сервера.

## Шаг 5. Подключить сайт
В `index.html` (на GitHub) впишите адрес `api.php`:
```js
const RSVP_ENDPOINT = "https://vmwed.ru/api/api.php";
```
Запушьте. Готово — заявки идут в Telegram, кнопка «📥 Скачать список гостей»
присылает .xlsx со столбцами: №, ФИО, Сможете прийти, Комментарий, Дата.

## Проверка
- `https://vmwed.ru/api/api.php` (GET) вернёт 405 — это норм (нужен POST).
- Заполните форму на сайте → в Telegram придёт уведомление.

> ⚠️ Если сайт на GitHub Pages, а `api.php` на reg.ru под тем же доменом vmwed.ru —
> это невозможно одновременно (домен указывает либо туда, либо туда). Тогда
> разместите PHP на поддомене, например `api.vmwed.ru`, и используйте
> `RSVP_ENDPOINT = "https://api.vmwed.ru/api.php"` (поддомен настраивается в DNS
> reg.ru записью A на 31.31.197.50).
